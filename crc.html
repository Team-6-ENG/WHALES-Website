Class: Main

Responsibility

  • Initialise and run the game lifecycle
  • Configure world and viewport dimensions
  • Load and manage shared graphical resources (SpriteBatch, fonts, UI skin)
  • Maintain global game configuration values and event locations
  • Store and manage the current player’s name
  • Set the initial screen and control screen transitions
  • Reset global game state between runs
  • Handle window resizing and resource disposal

Collaboration

  • HomeScreen - initial screen controlled through screen transitions
  • FitViewport, SpriteBatch, BitmapFont, LabelStyle, Skin - shared resources accessed by all screens
  • Event, HiddenEvent - global event state reset through Main

Class: Player

Responsibility

  • Manage the player character, including position, movement, and rotation
  • Handle input and apply movement physics, sprinting, wind, and woozy effects
  • Control player animations through the Animator superclass
  • Track and update energy levels, including sprinting and sugar crash effects
  • Render the player sprite on screen
  • Provide accessors and mutators for sprite, position, speed, and rotation
  • Reset player speed and manage shared movement state

Collaboration

  • Animator - inherited animation logic and frame control
  • Energy - stores and updates player energy
  • WorldScreen, HomeScreen - controlled by game screens for movement and events
  • Main - global game context, resource access

Class: Energy

Responsibility

  • Manage the player's energy level, including current energy and maximum energy
  • Regenerate energy over time when allowed
  • Provide methods to increase or reduce energy safely
  • Determine whether energy is accessible (usable) at the current state
  • Update energy each frame via tick() to apply regeneration
  • Render a visual energy label on the screen

Collaboration

  • Player - uses Energy to manage sprint and movement capability
  • Main.labelStyle - shared label style for consistent UI text

Class: Animator

Responsibility

  • Load a sprite sheet from a file
  • Cut the sprite sheet into frames based on rows and columns
  • Generate animations from sprite sheet frames and store them in a dictionary
  • Provide access to animations for other classes
  • Dispose of the sprite sheet texture when no longer needed

Collaboration

  • Player - uses Animator to generate movement and action animations

Class: WorldMap

  • Load the game map from JSON and store room data
  • Manage the current and previous room locations
  • Load and render room textures
  • Generate and manage obstacles, coins, and room borders
  • Handle player movement between rooms
  • Provide room information and access to obstacles

Collaboration

  • JsonRoom - stores room data loaded from JSON
  • Obstacle - represents walls, coins, and interactive objects in rooms
  • Main - provides constants like WORLD_WIDTH and WORLD_HEIGHT
  • OrthogonalDirection - indicates movement direction for player transitions

Class: JsonRoom

  • Represent a room as encoded in map.json
  • Store references to neighboring rooms in four directions: left, right, up, down
  • Store obstacles present in the room
  • Provide information about the next room in a given direction
  • Provide a readable string representation of the room for debugging

Collaboration

  • OrthogonalDirection - used to query neighboring rooms
  • JsonObstacle - represents obstacles within the room

Class: JsonObstacle

Responsibility

  • Represent an obstacle as encoded in map.json
  • Store position coordinates, width and height
  • Store texture path for the obstacle (textureInternalPath)
  • Provide a string representation for debugging purposes

Collaboration

  • WorldMap - used to load obstacles into a room
  • JsonRoom - contains an array of JsonObstacle objects

Interface: Event

Responsibility

  • Represent an event in the world that can be triggered
  • Track whether the event has been triggered
  • Provide methods to trigger the event and set/get its position
  • Maintain global counters for positive and negative events
  • Allow counters to be reset when the game resets

Collaboration

  • Any World Object or Manager that triggers events

Class: HomeScreen

Responsibility

  • Display the home screen to the player
  • Provide buttons for starting the game, viewing leaderboard, and achievements
  • Handle input events such as button clicks and Enter key press
  • Scale and position UI elements responsively based on screen size
  • Delegate screen transitions to the main game controller

Collaboration

  • Main - the main game controller that manages screen transitions
  • NameInputScreen - the screen to display when starting a new game
  • LeaderboardScreen - the screen to display when leaderboard is requested
  • AchievementScreen - the screen to display when achievements are requested

Class: Item

Responsibility

  • Represent an item in the game world that can be interacted with
  • Apply effect when triggered
  • Dispose itself after being used

Collaboration

  • Obstacle - inherits position, size, and collision behavior
  • ItemType - categorizes the item (e.g., consumable, inventory item)

Class: Frog

Responsibility

  • Represent a frog entity that can interact with the player by eating or releasing them
  • Manage animations for different frog states (idle, eyes, croak)
  • Track internal state such as whether it has swallowed the player, escape progress, and punch animation
  • Provide collision bounds for interaction with the player
  • Handle time-based updates for animations and state transitions

Collaboration

  • Animator - provides animation handling and sprite sheet processing
  • Player - interacts with frog during eat and escape sequences
  • Main - supplies global game state (e.g., current frog location)

Class: Security

  • Represent a campus security character
  • Track state such as whether it has stolen a duck and whether it is sleeping
  • Manage animations for different security states (walking, sleeping)
  • Update position
  • Provide collision bounds for interactions with other entities
  • Handle behavior on room changes (e.g., fall asleep if duck stolen)

Collaboration

  • Animator - provides animation handling and sprite sheet processing
  • WorldMap - provides room boundaries for movement limits
  • Main - provides global game state (e.g., current security location)

Class: SlipOnCoffee

Responsibility

  • Represent a negative event where the player slips on coffee
  • Track whether the event has been triggered and whether it is active
  • Control slip motion direction, duration, and rotation effects on the player
  • Reset itself when the player leaves the room
  • Increment the global negative event counter when triggered

Collaboration

  • Player - apply movement and rotation effects during slip
  • Event - increment negative event counter
  • Obstacle - provide bounding box and position

Class: DuckStatue

Responsibility

  • Manage all pieces of the duck statue in the game
  • Track which pieces have been found or triggered
  • Display jigsaw of pieces when found

Collaboration

  • DuckStatuePiece - represents individual parts of the duck
  • Main - provides world dimensions for positioning logic
  • WorldMap - provides boundary information for placing pieces

Class: DuckStatuePiece

Responsibility

  • Represent a single piece of the duck statue in the game
  • Track whether the piece has been collected (triggered)
  • Track whether the piece has been stolen by campus security
  • Provide information about which room the piece belongs to
  • Allow pieces to be stolen or returned (unsteal)

Collaboration

  • Obstacle - inherits position and size
  • DuckStatue - for assembling all pieces together
  • HiddenEvent - increments counter if the hat piece is collected
  • Main - for checking player location (e.g. security room logic)

Class: HiddenEvent

Responsibility

  • Represent a hidden event in the game
  • Track the event's position in the world
  • Track whether the event has been triggered or is active
  • Provide static counter for total hidden events triggered
  • Allow events to be triggered, reset, or deactivated

Collaboration

  • Event - implements the interface for triggering and tracking events
  • Other game systems - can query the counter or check if an event is active

Class: PowerCut

  • Represent a hidden event that darkens the player's screen for a fixed duration
  • Track whether the event has been triggered and whether it is active
  • Manage the darkness level over time, fading back to normal after the event ends
  • Reset itself when the power cut ends

Collaboration

  • WorldScreen - triggered through the main game loop

Class: RamblingProfessorEvent

Responsibility

  • Represent a hidden event of a professor character who continuously speaks lines of dialogue
  • Load and manage a list of speech lines from an external file
  • Update which line the professor is currently saying over time
  • Render the professor sprite and name label in the correct room
  • Track position and bounds for potential player interaction

Collaboration

  • HiddenEvent - Inherits functionality for triggering and activation tracking.
  • Main - Provides current world dimensions and room information for rendering and label positioning.

Class: WindEvent

Responsibility

  • Simulate wind along the main path of the game by moving and rotating leaves.
  • Trigger and manage wind direction and activity as a hidden environmental event.
  • Update leaf positions over time and remove leaves that go out of bounds.
  • Render leaves to the screen if the player is in the correct room.

Collaboration

  • HiddenEvent - Inherits functionality for triggering and activation tracking.
  • Main - Provides player location to determine if wind effects should be active.
  • AudioManager - Plays and stops wind sound based on wind activation and direction.

Class: NameInputScreen

Responsibilities

  • Collect the player's name and validate its length
  • Allow the player to select one of two character sprites
  • Dynamically position and scale UI elements according to screen size
  • Handle navigation actions, including starting the game or returning to the home screen

Collaboration

  • Main - stores the player's name, provides viewport and sprite batch, manages screen transitions
  • HomeScreen - used when the player navigates back
  • GameIntroScreen - launched after the player confirms name and character selection

Class: GameIntroScreen

Responsibilities

  • Provide an introductory narrative explaining the game context and objectives
  • Allow the player to proceed to the main game via a "Play" button
  • Dynamically scale and position UI elements according to screen size

Collaboration

  • Main - provides viewport, batch, label style, UI skin, and manages screen transitions
  • WorldScreen - launched when the player presses "Play" to start the game
  • WorldMap - provides boundary width for proper text layout

Class: WinScreen

Responsibilities

  • Display the winning screen background
  • Show player's final score and remaining time
  • Provide buttons to view the leaderboard, achievements, or return to the home screen
  • Save the player's score to the leaderboard upon first showing the screen
  • Scale and position UI elements dynamically based on screen size

Collaboration

  • Main - provides viewport, batch, label style, UI skin, and manages screen transitions
  • LeaderboardManager - handles saving and retrieving leaderboard scores
  • AchievementManager - tracks and provides player's achievements
  • LeaderboardScreen - displayed when the player clicks the "Leaderboard" button
  • AchievementScreen - displayed when the player clicks the "Achievements" button
  • HomeScreen - returned to when the player clicks the "Return" button

Class: AudioManager

Responsibilities

  • Manage all game sounds and background music
  • Provide methods to play individual sound effects
  • Control looping sounds such as background music and wind
  • Support volume control for sound effects and music separately
  • Enable muting and unmuting while preserving previous volume levels

Collaboration

  • Main game loop / Screens - call AudioManager methods to play sounds based on in-game events
  • Events and gameplay classes (e.g., WindEvent, PowerCut) - trigger specific sound effects

Class: Timer

Responsibilities

  • Keep track of the remaining game time in seconds
  • Update the timer each frame and manage countdown logic
  • Render the timer label and a "Game Over" timeout screen when time runs out
  • Support pausing and resuming the timer
  • Provide formatted time as mm:ss for display

Collaboration

  • Main - for label style and viewport/world dimensions

Class: Score

Responsibilities

  • Track the player's current score throughout the game
  • Provide methods to add or subtract points
  • Apply special bonuses, such as the duck bonus or time remaining bonus
  • Reset the score to zero when a new game starts
  • Render the score as a Label on the screen

Collaboration

  • Main - access label style and world dimensions for positioning the label

Class: LeaderboardManager

Responsibilities

  • Manage the local leaderboard by storing and retrieving player scores
  • Add new scores with player name, score, time left, and date
  • Sort scores in descending order5
  • Filter out incomplete or invalid entries when loading scores
  • Save and load scores from a local JSON file

Collaboration

  • ScoreEntry, LeaderboardData - represents individual scores and the full leaderboard
  • WinScreen, WorldScreen - calls addScore when a game ends

Class: LeaderboardData

Responsibilities

  • Encapsulates a list of leaderboard entries
  • Holds multiple ScoreEntry objects
  • Provides constructors for default initialization or from an existing list of scores

Collaboration

  • ScoreEntry - represents individual score records
  • LeaderboardManager - uses LeaderboardData to save/load scores to/from JSON

Class: ScoreEntry

Responsibilities

  • Encapsulate information about a single leaderboard record
  • Provide default values for backward compatibility
  • Ensure player name is always non-empty

Collaboration

  • LeaderboardManager - used to create, store, and sort leaderboard entries
  • LeaderboardData - stored as part of a list of ScoreEntry objects for JSON storage

Class: AchievementManager

Responsibilities

  • Initialize all available achievements with ID, name, and description
  • Unlock achievements and store the unlock date
  • Check if a specific achievement is unlocked
  • Reset all achievements
  • Save and load achievement data from local JSON storage
  • Provide access to all achievements or a specific achievement by ID

Collaborations

  • Achievement - represents an individual achievement and its state (unlocked, unlock date)
  • AchievementData - wrapper class for serializing/deserializing a list of Achievement objects to JSON
  • Game logic - interacts indirectly with gameplay systems to unlock achievements based on player actions

Class: Achievement

Responsibilities

  • Store a unique ID, name, and description for the achievement
  • Track whether the achievement has been unlocked
  • Store the date when the achievement was unlocked
  • Provide a method to unlock the achievement and set the unlock date

Collaborations

  • AchievementManager - creates, initializes, unlocks, and persists Achievement instances
  • AchievementData - wraps a list of Achievement objects for JSON serialization
  • Game logic - indirectly interacts with gameplay actions to unlock achievements

Class: AchievementData

Responsibilities

  • Store a list of Achievement objects for saving/loading from JSON
  • Provide a default constructor for JSON deserialization
  • Provide a constructor to initialize the list of achievements

Collaborations

  • AchievementManager - creates and loads AchievementData to save/load all achievements
  • Achievement - contains the individual achievements being stored

Class: AchievementNotification

Responsibilities

  • Display a notification message when an achievement is unlocked
  • Handle fade-in, display duration, and fade-out of the notification
  • Render notification text in world coordinates
  • Update the state of the notification each frame (timing and visibility)
  • Provide information about whether a notification is active

Collaborations

  • Achievement - provides the name and description for the notification text
  • Main - provides BitmapFont and world dimensions for rendering
  • WorldMap - provides boundary offset for positioning notification on screen

Class: AchievementScreen

Responsibilities

  • Display all game achievements (both unlocked and locked).
  • Show achievement name, status (UNLOCKED/LOCKED), description, and unlock date if available.
  • Provide navigation back to a previous screen (via "Back" button or ESC key).
  • Manage scaling and positioning of UI elements (labels and buttons) according to screen size.
  • Handle user input for UI interactions.

Collaborations

  • AchievementManager - retrieves the list of achievements and their status.
  • Achievement - provides data for each achievement (name, description, unlocked, date).
  • Main- access to game instance, viewport, font, UI skin, and world dimensions.
  • HomeScreen, returnScreen - used for returning to the previous screen.
